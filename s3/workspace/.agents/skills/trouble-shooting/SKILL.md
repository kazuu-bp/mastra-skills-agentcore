# トラブルシューティング: ファイル提供の失敗

ユーザーにうまくファイル（URLなど）を提供できていない場合、以下の仕組みと確認ポイントを参照して問題を解決してください。

## S3アップロードの仕組み
- 現在の仕様では、`outputs` フォルダ内にある **最新の1ファイルだけ** がS3にアップロードされ、そのファイルへのアクセスURLが発行される仕組みになっています。

## 確認ポイント
ファイルが正しく提供されない場合、または意図しないファイルが提供されてしまう場合、以下の点を確認してください。

1.  **ファイルが `outputs` フォルダ配下にあるか**
    提供したいファイルが確実に `outputs` フォルダ内に生成・配置されているかを確認してください。

2.  **他のファイルの方が新しくなっていないか**
    同じ `outputs` フォルダ内に、提供したいファイルよりも新しいタイムスタンプ（更新日時）を持つ別のファイルが存在している場合、その最新ファイルが誤ってアップロード・提供されてしまいます。
    **対処法**: 意図しない最新ファイルがある場合は、その一時的なファイルの名前を変更（退避）するか削除するなどして、提供したいファイルが最新の状態になるように調整してください。
